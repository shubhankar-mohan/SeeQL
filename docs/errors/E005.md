# E005 — Required Cloud SQL flag is missing or wrong

**Severity:** blocking on GCP Cloud SQL. Irrelevant on self-hosted MySQL.

## What this means

SeeQL needs four flags set on the MySQL instance to function fully:

| Flag | Required value | What breaks without it |
|------|----------------|------------------------|
| `performance_schema` | `on` | Most collectors (see [E002](E002.md)) |
| `slow_query_log` | `on` | Slow query log collector |
| `long_query_time` | `1` (or lower) | Slow queries aren't caught |
| `innodb_monitor_enable` | `all` | InnoDB metric counters empty |

Cloud SQL requires an instance restart after changing these flags.

## How to verify

```sql
SHOW VARIABLES WHERE Variable_name IN (
  'performance_schema', 'slow_query_log',
  'long_query_time', 'innodb_monitor_enable'
);
```

All four should show the expected values in the result set.

## How to fix

**Console:** Cloud SQL instance → Edit → Flags and parameters → add the
four flags.

**gcloud CLI:**

```bash
gcloud sql instances patch <instance> \
  --database-flags=performance_schema=on,slow_query_log=on,long_query_time=1,innodb_monitor_enable=all
```

Cloud SQL schedules a restart after the patch. The instance is
unavailable for ~30 seconds.
