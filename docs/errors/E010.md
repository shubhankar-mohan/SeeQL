# E010 — Invalid time range for replay

**Severity:** CLI-only. Doesn't affect the running service.

## What this means

`seeql replay --from X --to Y` received timestamps that SeeQL can't
reconcile:

- One or both aren't ISO 8601
- `--to` is earlier than `--from`
- The range is zero-width (`from == to`)

## How to verify

```bash
# ISO 8601 looks like one of these:
seeql replay --from 2026-04-10T03:00:00 --to 2026-04-10T04:00:00
seeql replay --from 2026-04-10T03:00:00+00:00 --to 2026-04-10T04:00:00+00:00
seeql replay --from "2026-04-10 03:00:00" --to "2026-04-10 04:00:00"   # also accepted
```

## How to fix

- Use ISO 8601 strings (`YYYY-MM-DDTHH:MM:SS` or with a timezone
  suffix). If you omit the timezone, SeeQL treats it as UTC.
- Make sure `--to` is *after* `--from`.
- For at-least-1-minute windows if you want any events to show up.

## Easier alternatives

For auto-detected windows:

```bash
seeql replay --latest              # most recent incident
seeql replay --incident 42         # by id (see `seeql incidents list`)
```

`seeql incidents list` shows ids, timestamps, severities, and involved
metrics — pick one and replay it without manually picking a range.
