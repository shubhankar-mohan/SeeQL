# E007 — Permission denied (missing grants)

**Severity:** blocking. Different from [E001](E001.md) — the login
itself works, but required reads fail.

## What this means

`dba_agent` connected successfully but can't read the tables SeeQL
needs. Common misses:

- `PROCESS` privilege missing (can't see other users' threads in
  `SHOW PROCESSLIST` / `performance_schema.threads`)
- `SELECT` missing on `performance_schema.*`
- `SELECT` missing on `information_schema.INNODB_METRICS` (requires
  `SELECT` on `performance_schema` specifically)

## How to verify

```sql
-- What grants does dba_agent have right now?
SHOW GRANTS FOR 'dba_agent'@'%';
-- Expect:
-- GRANT SELECT, PROCESS ON *.* TO `dba_agent`@`%`
```

## How to fix

```sql
GRANT SELECT, PROCESS ON *.* TO 'dba_agent'@'%';
FLUSH PRIVILEGES;
```

That's all SeeQL needs — read-only across every database plus the
ability to see other users' running queries. No `SUPER`, no writes.

Then re-run:

```bash
seeql doctor
```
