# E003 — GCP Application Default Credentials not configured

**Severity:** optional. You see this only if you're trying to use GCP
features (Cloud Monitoring, Cloud Logging, Vertex AI).

## What this means

SeeQL tried to authenticate to Google Cloud APIs and couldn't find
credentials. ADC discovery checks, in order:

1. `GOOGLE_APPLICATION_CREDENTIALS` env var (path to a service account
   JSON)
2. `MONITORING_APPLICATION_CREDENTIALS` env var (SeeQL-specific, overrides
   the above for monitoring/logging only)
3. `gcloud auth application-default login` credentials on disk
4. GCE metadata server (when running on a GCE VM)

If none match, the GCP collectors skip silently and the GCP-backed LLM
backends (Gemini, Vertex-Claude) fail with E003.

## When you'll see it

- Medium-loop cycle: warnings from `gcp_metrics` and `gcp_slow_log` that
  creds couldn't be resolved (these collectors skip gracefully, no error
  raised)
- `seeql serve` with `agent.enabled: true` and a `gemini-*` or
  `claude-*-vertex` model configured

## How to fix

**Local dev:**

```bash
gcloud auth application-default login
```

**GCE VM (production):** attach a service account to the VM with the
roles SeeQL needs:

- `roles/monitoring.viewer` — Cloud Monitoring
- `roles/logging.viewer` — Cloud Logging
- `roles/aiplatform.user` — Vertex AI (optional, only for LLM)

**Docker:** mount a service account JSON and point
`GOOGLE_APPLICATION_CREDENTIALS` at it:

```bash
docker run ... \
  -v /path/to/sa.json:/app/gcp-sa.json:ro \
  -e GOOGLE_APPLICATION_CREDENTIALS=/app/gcp-sa.json \
  ghcr.io/shubhankar-mohan/seeql:latest-gcp
```

## Running without GCP

SeeQL works fine without any GCP wiring — use the generic image (not
`-gcp`), set `agent.enabled: false` or configure Claude via Anthropic
API key (`ANTHROPIC_API_KEY`). The GCP collectors simply don't run.
