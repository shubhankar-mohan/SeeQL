# E004 — Invalid config file

**Severity:** blocking. SeeQL refuses to start with a bad config.

## What this means

`config/settings.yaml` or `settings.local.yaml` failed to load. The
most common causes:

1. YAML syntax error (tabs instead of spaces, unclosed quotes, bad
   indentation)
2. Required top-level key missing (e.g. no `production_db` block)
3. Environment variable placeholder (`${FOO}`) didn't resolve because
   `FOO` isn't exported

## When you'll see it

- Immediately on `seeql check`, `seeql run`, `seeql serve`, `seeql doctor`
- The process exits non-zero before any collection starts

## How to verify

```bash
# Plain YAML parse
python -c "import yaml; yaml.safe_load(open('settings.local.yaml'))"

# See which env vars are referenced
grep -o '\${[A-Z_]*}' settings.local.yaml | sort -u

# Check they're all exported
env | grep -E "PROD_DB_|ANTHROPIC_|SLACK_|GCP_"
```

## How to fix

**YAML syntax error:** the Python traceback names the line and column.
Use a YAML linter (`yamllint settings.local.yaml`) for a fuller report.

**Missing key:** compare your file against the stock
[`config/settings.yaml`](../../config/settings.yaml). Copy anything
missing and override only what you need.

**Unresolved `${VAR}`:** export the env var before launching SeeQL, or
put the literal value in `settings.local.yaml` (only for non-secret
config; never commit passwords).
