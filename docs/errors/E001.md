# E001 — MySQL authentication failed

**Severity:** blocking. SeeQL cannot collect anything without a working
MySQL connection.

## What this means

The monitoring user (`dba_agent` by default) cannot log in to the
configured production MySQL host. Either the password is wrong, the
user doesn't exist yet, or it exists but isn't allowed from the host
SeeQL is running on.

## When you'll see it

- At startup, during the first collector cycle
- From `seeql check` and `seeql doctor`
- In the `/health` endpoint as `status: degraded`

## How to verify

```bash
# Does the user exist?
mysql -h <host> -u <root_or_admin_user> -p \
  -e "SELECT user, host FROM mysql.user WHERE user='dba_agent';"

# Can we log in as dba_agent from this machine?
mysql -h <host> -u dba_agent -p -e "SELECT 1;"
```

## How to fix

Create the user with the grants SeeQL needs:

```sql
CREATE USER 'dba_agent'@'%' IDENTIFIED BY '<strong_password>';
GRANT SELECT, PROCESS ON *.* TO 'dba_agent'@'%';
FLUSH PRIVILEGES;
```

The `'@'%'` host wildcard is fine for most setups. For tighter
security, restrict to SeeQL's specific host:

```sql
CREATE USER 'dba_agent'@'10.0.1.42' IDENTIFIED BY '...';
GRANT SELECT, PROCESS ON *.* TO 'dba_agent'@'10.0.1.42';
```

Then set `PROD_DB_PASSWORD` (env var) or `production_db.password`
(`settings.local.yaml`) to the password you used.
